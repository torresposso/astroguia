# Shared working memory for multi-agent coordination

Use Mastra working memory with `scope: 'resource'` and `resourceId = personId` so all 7 agents share a person's birth data (date, time, coordinates, timezone, house system) without each specialist querying the database independently. The orchestrator writes person data to working memory via `load_person`; all 6 specialists are configured with `readOnly: true` on their working memory — they can read the birth data but cannot modify it. All agents share the same LibSQL storage backend so the working memory keyed by `resourceId` is visible across all threads for the same person. The alternative (passing birth data as explicit parameters in each delegation via the supervisor's `onDelegationStart` hook with `modifiedPrompt`) would be more explicit but more verbose, would couple the supervisor to the data schema, and would require re-sending data on every delegation.
